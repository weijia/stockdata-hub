## 费用图例

- 🆓 **免费 / 接近免费**：可直接使用，无付费门槛（或仅需自建服务器等一次性运维，数据本身免费）
- 🔑 **需 Token**：通常需注册账号 / 商业授权（iTick、tushare 等）
- 🔒 **单独开权限 / 付费**：实时类高级数据，需单独申请开通（多为付费项）
- ❌ **不可用**：当前被墙或无法访问

> **已确认免费或接近免费的来源**：🆓 **腾讯、新浪、通达信（mootdx）、openstockdata、tushare（基础行情）**。
> 付费 / 受限：**iTick（🔑）、tushare 实时接口（🔒）、东财（❌ 被墙）**。
> 注：**akshare 是开源聚合接口层，本身不生产数据**，其所有调用穿透后都落在「东财 / 新浪 / 腾讯」三源上（如股票日K、港股、代码列表走东财，ETF 回退走新浪，部分日K走腾讯）。由于这些真实源已在上表直接列出，且 akshare 走东财的部分同样被墙，**故文档不单列 akshare，相关数据类型直接标注底层真实源**。

## 三、附：分时 vs 实时报价的区别

两者是**「时间序列」vs「单点汇总」**的关系——分时图是最细的原始数据，实时报价是从它（或日K）聚合出来的当前快照。

| | 股票分时（分时图，含均价） | 实时报价 / 快照 |
|---|---|---|
| **形态** | 一整天的**时间序列**：每个时间点一行 | 当前时刻的**单个汇总行** |
| **一行内容** | `(时间, 价格, 成交量, 均价)` | `(最新价, 今开, 昨收, 最高, 最低, 总成交量, 总成交额)` |
| **数据量** | 当天约 240 个点（每分钟一个） | 永远 1 行 |
| **均价** | **含**（黄色均价线 = 累计均价） | 一般不含均价字段 |
| **用途** | 画分时走势图、看盘中轨迹、做分时趋势 | 自选股列表/监控里显示"现在多少钱、涨跌幅多少" |
| **更新方式** | 每分钟追加一个新采样点 | 每次刷新返回"此刻的最新状态" |

**关键关系**：实时报价的「最新价」就是分时序列的最后一点；今开 / 最高 / 最低 / 成交量都是对当天分时序列的聚合。即**分时是最细原始数据，实时报价是它的"当前摘要"**。

## 四、一句话总结

- **K线类（日 / 分钟）**：主力可用源为 **通达信🆓 / 新浪🆓 / 腾讯🆓**；东财被墙。
- **实时价**：根数据源是 **腾讯🆓**（免费可达），通达信🆓分时为补充；tushare 实时接口需🔒单独开权限/付费，仅作备选。
- **周边数据（资金流 / 龙虎榜 / 分红 / 大宗 / 全市场快照）**：多依赖**东财且已被墙**，属当前缺口（其中龙虎榜、分红有新浪🆓可选）。
- **批量查询多只**：✅ 支持的有 **腾讯（首选）、新浪、东财（被墙）、iTick**；⚠️ 通达信仅 quotes 实时接口可批量；❌ **tushare 不支持批量**（需逐只循环），openstockdata 未确认。实时批量报价接口依赖腾讯 / 通达信 quotes 的批量能力。
